# Aperio

> **Aperio** /ah-PEH-ree-oh/ — Latin: *I open. I reveal.*
>
> A language whose programs are **lotuses** — structures that grow
> themselves, audit their own correctness, and dissolve cleanly when
> their work is done.

These are the Aperio docs. They live as a single book with five
doorways. Pick whichever fits how you read.

## The doorways

- **[Quickstart](./quickstart/introduction.md)** — five minutes.
  Install the toolchain, run a one-locus program, see where to
  read next. Start here if you want to verify Aperio works on your
  machine before committing to anything longer.
- **[The Grimoire](./grimoire/introduction.md)** — the magical
  onboarding path. Arrival, reveal, vocabulary, emergence — the
  four moments of opening a lotus. Start here if you prefer
  narrative to specification.
- **[The Aperio Programming Language](./book/introduction.md)** —
  the technical book. A layered substrate-up tutorial, shaped like
  the Rust Book. Start here if you've used Rust or Go and want a
  systems-language reading path.
- **[The Reference](./reference/introduction.md)** — the formal
  grammar, semantics, and conventions. Pages follow a strict
  Synopsis / Grammar / Semantics / Examples / See Also template.
  Read this when the tutorial has gestured at a thing and you want
  the precise rule.
- **[The Standard Library](./std/roadmap.md)** — what ships in
  `std::*`, in active development. Phase 1 modules are usable
  today; later phases are scheduled.

The grimoire and the technical book teach the same material in
different registers. The reference and stdlib pages are the
destination either way.

## Status

Aperio is in active development. These docs are local-only until
a public v1 release. The CI workflow validates that `mdbook build`
succeeds; nothing is published yet.

The language is shaped by lessons absorbed from an earlier
exploratory project — *the ancient texts* — but the surface area
described here stands on its own. Where the docs assume prior
art, they spell it out.
