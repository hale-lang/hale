- `[#]` **3. Recognition projection class proper backing** — DEFERRED, design depth
  - C runtime: implement `lotus_recpool_*` family — bitmap-backed
    fixed-cap pool, no malloc in steady state.
  - Codegen: route recognition class to `lotus_recpool_*` instead
    of `lotus_arena_create_subregion`.
  - **Open design question that needs user resolution before impl:**
    Recognition pools are fixed-size cells. Child loci accepted by
    a recognition-class parent can't grow their own arena beyond
    the fixed cell size. spec/memory.md says recognition is for
    "summary-only per coordinatee; minimal per-coordinatee state"
    — but the existing v0 codegen still allocates a per-child
    `lotus_arena_t*` that can grow chunks dynamically. To make
    recognition genuinely zero-malloc-steady-state, either:
    (a) recognition children get a fixed-size cell that their
        arena allocations into the cell's bytes — requires a new
        arena variant `lotus_arena_t_fixed` that takes a backing
        slab + cap.
    (b) recognition children skip the arena entirely and any
        composite-literal allocations spill to a fallback (which
        defeats the "no malloc" promise).
    (c) accept-time cap-overflow falls back to malloc with a
        warning (graceful degradation).
  - Sizing question: parent annotation `: projection recognition(cap=256)`
    or a default? cap is a user-visible decision.
  - Recommendation: defer until a workload exercises it. Per
    spec, "spec/memory.md flags the gap" and "Functionally
    equivalent to chunked at this layer" so v1 users don't
    actually see a behavior change today.

- `[~]` **4. Slot 1..N parent-override (`as_parent_for ChildL`)** —
  language surface only (v1.x-4); runtime mechanic = v1.x-4b
  - DONE: AST CapacitySlot.as_parent_for: Option<Ident>; lexer
    AsParentFor token; parser trailing clause; typecheck
    validates ChildL exists + has matching slot name.
  - DONE: codegen explicitly REJECTS as_parent_for with "v1.x-4b
    pending" diagnostic so users don't get silent miscompilation.
  - DONE: grammar.ebnf updated with the optional trailing clause.
  - 3 tests in f22_as_parent_for_surface.rs (parses + codegen
    rejects, missing-locus rejected, mismatched-slot rejected).
  - **v1.x-4b TODO (runtime mechanic):**
    - Add `__slot_borrowed_mask: i64` field to every locus struct
      (uniform layout). Init to 0.
    - At child instantiation when parent_accepts_us AND parent
      has `as_parent_for ThisLocus` for a slot matching child's
      slot by name: load parent's __slot_<n>, store into child's
      __slot_<n>, OR-in the matching bit in __slot_borrowed_mask.
    - At emit_locus_arena_destroy: for each slot, check the bit.
      Skip destroy on borrowed slots — destroying them would
      corrupt the parent's allocator (children dissolve before
      parents per F.4 depth-first cascade).
    - Also: extend symbol-level LocusInfo with slot kind+elem_ty
      so typecheck can validate kind+ty match (not just name).

## Form-extending (Design mechanics at new levels)

- `[ ]` **6. Result + `?` operator**
  - Type: `Result<T, E>` as a built-in enum (or stdlib type with
    compiler awareness).
  - Lexer/parser: `?` postfix operator.
  - Codegen: `?` lowers to "match on result; Ok unwraps; Err early-
    returns from the enclosing fn" — fn must declare `-> Result<_, _>`.
  - Typecheck: `?` only valid in fns returning Result.
  - Failure-propagation-upward at value-shape — currently this
    mechanic exists only at locus-level via `bubble`/`on_failure`.

- `[ ]` **7. Match exhaustiveness via case folding**
  - check.rs `match_is_exhaustive`: extend to handle
    `0..=N` ranges, enum variants whose payloads have known
    constructor patterns, and folded-equivalent cases.

- `[ ]` **8. `type` records hold `fn(...)` fields**
  - Friction: 2026-05-11 `type-records-cannot-hold-fn-pointer-fields`
  - Codegen: `synthesize_serializer` + struct field layout: handle
    `CodegenTy::FnPtr { .. }` as a `ptr` field (it already is at
    LLVM level; just needs the right ser/deser handling — or reject
    for bus payloads but accept in pure struct fields).
