## Locked-in design decisions (don't relitigate)

- **Four sub-modes for recognition class**, per design
  2026-05-11:
  - `fixed_cell(bytes=N)` — each child fits in N bytes;
    overflow is a hard runtime error. Bounded state.
  - `spillover(bytes=N)` — fixed N-byte cell + malloc fallback
    + one-time warning. Graceful degradation.
  - `summary_only` — children carry zero per-instance state;
    all allocations live in the parent's arena.
  - `shared_slab(bytes=N)` — one bump arena of N bytes serves
    all children; per-child dissolve is no-op; whole slab frees
    at parent dissolve.
- **v1 ships `fixed_cell` + `shared_slab`.** `spillover` +
  `summary_only` parse + typecheck but reject at codegen with
  "v1.x pending" — mirrors the v1.x-4 surface-only pattern.
- **No default sub-mode.** Bare `: projection recognition`
  becomes a parse error. The user MUST commit to a sub-mode at
  the declaration site. Forcing function is productive — same
  shape as the two-channel rule's force-the-user-to-name-the-
  channel discipline locked in 2026-05-12.
- **Syntax** (proposed; finalize in PR1):
  ```
  : projection recognition(cap=N, fixed_cell(bytes=K))
  : projection recognition(cap=N, shared_slab(bytes=K))
  : projection recognition(cap=N, spillover(bytes=K))
  : projection recognition(cap=N, summary_only)
  ```
  `cap=N` is the child-count cap (existing semantics).

## Work breakdown (in execution order)

1. **C runtime: `lotus_recpool_fixed_*` + `lotus_recpool_slab_*`.**
   - `lotus_recpool_fixed_create(cap_count, cell_bytes)` →
     allocates header + bitmap + `cap_count × cell_bytes`
     contiguous slab. Returns recpool handle.
   - `lotus_recpool_fixed_acquire(recpool)` → bitmap scan for
     first-free slot; returns ptr to cell or null on full.
   - `lotus_recpool_fixed_release(recpool, cell_ptr)` →
     compute slot idx from ptr offset; clear bit.
   - `lotus_recpool_fixed_destroy(recpool)` → free header.
   - `lotus_recpool_slab_*` similar shape but no per-cell
     tracking — single bump pointer + child-boundary bitmap for
     parent-dissolve teardown. Per-child release is no-op.
   - Implementation lives in `crates/aperio-codegen/runtime/
     lotus_arena.c` (or a fresh `lotus_recpool.c` — decision
     in PR1).
   - ~200-300 lines total. Standalone — no codegen integration
     yet.

2. **Parser + AST.** Extend `projection_class` to carry
   sub-mode + params.

## Two-channel rule reference

The 2026-05-12 lock-in: `fallible(E)` permitted only on free fns
and stdlib-synthesized `@form(...)` methods; rejected on user-
declared locus methods. Closure-violation channel and value-
error channel meet only at the implicit main locus root via
`lotus_root_panic`. See `spec/semantics.md` § "Fallible call
semantics" § "Where each channel lives" for canon.

This same shape — "force the user to name the channel/sub-mode
at the declaration site" — is what v1.x-3's no-default-sub-mode
rule expresses for projection class. Coherent design discipline
across both surfaces.
