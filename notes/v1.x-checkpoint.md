## Form-shaping (substrate-honesty gaps)

- `[x]` **1. Interpreter parity for F.22 capacity slots** — commit TBD
  - Value::Cell + LocusHandle.slots + SlotState (Pool/Heap)
  - eval.rs instantiate_locus walks LocusMember::Capacity
  - try_eval_capacity_slot_call intercepts self.X.method() shape
  - tests/f22_capacity_interpreter.rs (5 tests; round-trip, churn,
    cross-kind rejection, multi-slot coexist)
  - 439 / 0 workspace test count

- `[ ]` **2. Cell content I/O**
  - Syntax design question first: `*cell` deref vs `cell` auto-load
    vs `cell.field` for struct cells (analogous to recv-self field
    access). Probably `cell.field` for struct cells and reject
    primitive cells until clearer surface.
  - Codegen: extend `lower_expr` for `Expr::Field { receiver, name }`
    when `receiver_ty` is `CodegenTy::Cell(_)` — GEP through the
    pointer + load.
  - Codegen: `Stmt::Assign` LValueSeg for cell-field write — GEP +
    store. Compound `+=` deferred.
  - Typecheck: `Cell<T>.field` resolves like `T.field`.

- `[ ]` **3. Recognition projection class proper backing**
  - C runtime: implement `lotus_recpool_*` family — bitmap-backed
    fixed-cap pool, no malloc in steady state. Sizing comes from
    parent-locus annotation or stats.
  - Codegen: route recognition class to `lotus_recpool_*` instead
    of `lotus_arena_create_subregion`.
  - Test: recognition-class accept/dissolve cycles don't malloc
    after warmup.

- `[ ]` **4. Slot 1..N parent-override (`as_parent_for ChildL`)**
  - AST: `CapacitySlot` carries `Option<as_parent_for_locus: Ident>`.
  - Lexer: `as_parent_for` keyword (compound).
  - Parser: trailing `as_parent_for ChildL` clause on slot decl.
  - Typecheck: ChildL must exist + must have a slot of matching
    name/kind/elem_ty.
  - Codegen: child's `__slot_<name>` initialized to parent's
    `__slot_<name>` value instead of fresh allocator.
  - Child's dissolve skips destroy on parent-overridden slots.

- `[ ]` **5. Slot-of-origin tracking on `Cell<T>`**
  - CodegenTy: `Cell<T>` carries slot identity (locus_name, slot_name)
    in addition to elem_ty. New variant or extended box.
  - Typecheck: `release(c)` / `free(c)` checks the cell came from
    *this* slot, not just `Cell<T>` shape.
  - Diagnostic: span-targeted "cell came from slot X, can't release
    into slot Y".

## Form-extending (Design mechanics at new levels)

- `[ ]` **6. Result + `?` operator**
  - Type: `Result<T, E>` as a built-in enum (or stdlib type with
    compiler awareness).
  - Lexer/parser: `?` postfix operator.
  - Codegen: `?` lowers to "match on result; Ok unwraps; Err early-
    returns from the enclosing fn" — fn must declare `-> Result<_, _>`.
  - Typecheck: `?` only valid in fns returning Result.
  - Failure-propagation-upward at value-shape — currently this
    mechanic exists only at locus-level via `bubble`/`on_failure`.

- `[ ]` **7. Match exhaustiveness via case folding**
  - check.rs `match_is_exhaustive`: extend to handle
    `0..=N` ranges, enum variants whose payloads have known
    constructor patterns, and folded-equivalent cases.

- `[ ]` **8. `type` records hold `fn(...)` fields**
  - Friction: 2026-05-11 `type-records-cannot-hold-fn-pointer-fields`
  - Codegen: `synthesize_serializer` + struct field layout: handle
    `CodegenTy::FnPtr { .. }` as a `ptr` field (it already is at
    LLVM level; just needs the right ser/deser handling — or reject
    for bus payloads but accept in pure struct fields).
