# language/

Aperio as a programming language. The technical track.

**What goes here.** Substrate-up tutorial, reference grammar +
semantics + types + runtime, stdlib catalog, working examples.
The shape `book/` + `reference/` + `std/` used to take in the
archive, rebuilt against the current spec.

**Reader.** Someone who wants to write Aperio. Came in cold,
probably fluent in Rust or Go or Zig, wants the language
proper without the mythos overlay.

**Source of truth.** `../../spec/` is canonical. Pages here
teach from the spec; they don't duplicate it.

**Status.** Empty scaffold. Content arrives in purpose-driven
sessions.

**Worth reading first** (when this track starts being filled):

- `../../spec/` — grammar, semantics, types, runtime, stdlib,
  forms, design rationale. The spec was swept pristine on
  2026-05-12 specifically for this rebuild.
- `../../spec/styleguide.md` — the six-pattern catalog +
  naming + composition rules.
- `../../notes/aperio-language-shape.md` — the five-axis
  orientation (`params` / `capacity` / `projection` /
  `schedule` / `@form`) and how they compose.
- `../archive/docs/src/book/` and `../archive/docs/src/reference/` —
  the old versions of this track, useful for outline and
  examples but stylistically and substantively stale.
